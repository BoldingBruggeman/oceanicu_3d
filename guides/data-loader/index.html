Data Loader Guide#

Overview#

data-loader (entry point for cli/data_loader.py) is the unified command for fetching and inspecting ocean data from multiple sources:

  • Local NetCDF — single file, glob pattern, or directory (auto-concatenated along time)
  • Local / cloud Zarr — local .zarr stores or cloud URIs (s3://, gs://, az://)
  • Copernicus Marine (CMEMS) — live remote download via copernicus:// URI
  • CMIP6 (PANGEO / intake-esm) — via cmip6:// URI
  • ISIMIP — via isimip:// URI
  • ERA5 cloud — ARCO-ERA5 on Google Cloud (era5gcs://) or ERA5-PDS on AWS (era5aws://)
  • Argo floats — Ifremer GDAC ERDDAP (argo://)
  • ICES Oceanographic Database — via ices:// or ices-fp://
  • EMODnet Physics — via emodnet://

All sources support domain and time-range subsetting. Loaded data is returned as an xr.Dataset (or xr.DataArray when a single variable is requested).

CLI usage (data-loader)#

Local files#

# Show metadata only
data-loader /data/model.nc --info

# Load and print summary; save to NetCDF
data-loader /data/model.nc --variable sst --output sst.nc

# Domain and time subsetting (lon_min lon_max lat_min lat_max)
data-loader /data/model.nc --variable sst \
    --domain -5. 15. 50. 65. --time-range 2020-01-01 2020-12-31 \
    --output sst_subset.nc

Copernicus Marine (CMEMS)#

# Describe a dataset without downloading
data-loader --cmems-describe cmems_obs-sst_atl_phy_nrt_l3s_P1D-m

# Browse catalogue, optionally filter by keywords
data-loader --cmems-catalog --output cmems_catalog.json
data-loader --cmems-catalog --contains SST North Atlantic

# Download with subsetting
data-loader --copernicus cmems_obs-sst_atl_phy_nrt_l3s_P1D-m \
    --variable sea_surface_temperature \
    --domain -5. 15. 50. 65. \
    --time-range 2023-01-01 2023-01-31 \
    --output sst_jan2023.nc

CMIP6 (PANGEO)#

# Browse / filter the catalog
data-loader --cmip6-describe --source-id CESM2 --variable-id tos
data-loader --cmip6-catalog --output pangeo_cmip6.json

# Load a subset  (SOURCE/EXPERIMENT/VARIABLE/TABLE/MEMBER[/GRID])
data-loader --cmip6 CESM2/historical/tos/Omon/r1i1p1f1 \
    --domain -5. 15. 50. 65. --time-range 1990-01-01 1990-12-31 \
    --output cesm2_tos.nc

ERA5 from cloud (no CDS account required)#

# List available variable mappings
data-loader --era5-varlist gcs
data-loader --era5-varlist aws

# Load from Google Cloud (ARCO-ERA5 Zarr, anonymous)
data-loader --era5gcs rsds \
    --domain -20. 30. 45. 75. --time-range 2000-01-01 2000-12-31 \
    --output era5_rsds_2000.nc

# Override automatic ERA5 variable-name mapping
data-loader --era5gcs rsds --era5-name surface_solar_radiation_downwards \
    --domain -20. 30. 45. 75. --time-range 2000-06-01 2000-06-30 \
    --output era5_rsds_jun2000.nc

# Override the ARCO store URL
data-loader --era5gcs tas \
    --era5-store gs://gcp-public-data-arco-era5/ar/1959-2022-full_37-1h-0p25deg-chunk-1.zarr-v2 \
    --time-range 1990-01-01 1990-12-31 --output tas_1990.nc

# Load from AWS S3 (ERA5-PDS, anonymous)
data-loader --era5aws tas \
    --domain -5. 15. 50. 65. --time-range 2010-01-01 2010-03-31 \
    --output era5_tas_q1_2010.nc

Argo floats#

# List parameter catalogue
data-loader --argo-catalog

# Load physical profiles for a domain
data-loader --argo physical \
    --domain -5. 15. 50. 65. --time-range 2022-01-01 2022-12-31 \
    --output argo_ns_2022.nc

# Load specific parameters, restrict to float WMO numbers
data-loader --argo TEMP,PSAL --wmo 6903548 6903549 --output argo_floats.nc

ICES Oceanographic Database#

data-loader --ices-catalog
data-loader --ices physical \
    --domain -5. 15. 50. 65. --time-range 2015-01-01 2020-12-31 \
    --output ices_ns.nc

YAML config#

data-loader --config examples/data_loader_examples.yaml --section sst_copernicus

Loading local files#

NetCDF#

NetCDFLoader (in lib/netcdf_loader.py) handles single files, glob patterns, and directories. When a directory is given, all .nc files are found and concatenated along the time dimension (sorted by filename).

Triple-fallback time decoder — applied in order until one succeeds:

  1. Standard xr.open_dataset with default time decoding
  2. cftime-based decoding (for proleptic_gregorian and other non-standard calendars)
  3. netCDF4.num2date direct conversion (for masked fill-value time axes and other broken encodings)

Kelvin → Celsius auto-conversion — if a variable’s units attribute is "K" or "kelvin" and its values are plausibly in the Kelvin range (> 200 K), the loader subtracts 273.15 and updates the units to "°C" automatically.

Singleton squeezing — after loading, dimensions of size 1 that are not time/latitude/longitude are dropped from the returned DataArray so downstream code does not need to handle them.

Variable resolution#

When a specific variable is requested, DataLoader._extract_variable() tries:

  1. Exact name match
  2. Only variable in dataset (single-variable file)
  3. Alias lookup via CoordinateMapper (reads config/name_mappings.yaml)

The built-in fallback table maps common short names to known aliases:

RequestedAlso matches
ssttemp, temperature, tsurf, analysed_sst
sshzeta, eta, elevation, sea_surface_height
ssssalt, salinity, ssurf, analysed_sss
temptemperature, potential_temperature, theta
saltsalinity, practical_salinity

Directory loading#

When a directory path is given to load_model_data():

  1. Looks for .nc files in the directory (then recursively if none found)
  2. If a variable is specified, prefers files whose stem contains the variable name
  3. Multiple matching files are concatenated along time and sorted

Copernicus Marine#

Requires the copernicusmarine package:

pip install copernicusmarine

Authentication uses the standard copernicusmarine credential store (run copernicusmarine login once to save credentials).

Key parameters passed to CopernicusLoader.load():

  • dataset_id — the CMEMS dataset identifier (e.g. cmems_obs-sst_atl_phy_nrt_l3s_P1D-m)
  • variables — list of variable names to download
  • domain — bounding box dict with lon_min, lon_max, lat_min, lat_max
  • time_range — (start, end) strings forwarded directly to the CMEMS API
  • minimum_depth / maximum_depth — optional depth range (metres)
  • service — optional service override (e.g. arco-geo-series)

The --cmems-describe flag calls CopernicusLoader.describe() and prints available datasets and services without downloading any data.

CMIP6 / PANGEO#

Requires intake, intake-esm, and gcsfs:

pip install intake intake-esm gcsfs

Datasets are discovered via the PANGEO CMIP6 Zarr catalog at https://storage.googleapis.com/cmip6/pangeo-cmip6.json.

The URI format is:

cmip6://SOURCE_ID/EXPERIMENT_ID/VARIABLE_ID/TABLE_ID/MEMBER_ID[/GRID_LABEL]

Example: cmip6://CESM2/historical/tos/Omon/r1i1p1f1

After loading, DataLoader automatically:

  • Sorts the time axis (intake-esm may concatenate chunks with inconsistent offsets)
  • Normalises longitude from 0..360 to −180..180 if needed

Results can be cached as compressed NetCDF by passing cache_dir to DataLoader:

loader = DataLoader(cache_dir='/data/cmip6_cache')
ds = loader.load_data('cmip6://CESM2/historical/tos/Omon/r1i1p1f1',
                      domain={'lon_min': -5, 'lon_max': 15,
                              'lat_min': 50, 'lat_max': 65},
                      time_range=('1990-01-01', '1990-12-31'))

Zarr stores#

Dependencies#

pip install zarr            # required for any Zarr
pip install s3fs            # S3 (AWS)
pip install gcsfs           # Google Cloud Storage
pip install fsspec          # HTTP/HTTPS

Local Zarr#

Auto-detected when the path ends in .zarr:

ds = loader.load_data('/data/ocean/model.zarr', variable='sst')

Cloud Zarr#

Cloud paths (s3://, gs://, az://) are also detected as Zarr automatically. Pass credentials via storage_options:

# Public S3 bucket
ds = loader.load_data(
    's3://public-ocean-data/era5.zarr',
    variable='sst',
    storage_options={'anon': True},
)

# GCS with service account
ds = loader.load_data(
    'gs://ocean-data-bucket/model.zarr',
    variable='sst',
    storage_options={'token': '/path/to/credentials.json'},
)

Chunking#

Pass a chunks dict or 'auto' to open the store with Dask lazy loading:

ds = loader.load_data(
    's3://bucket/ocean.zarr',
    variable='sst',
    chunks={'time': 30, 'lat': 200, 'lon': 200},
    storage_options={'anon': True},
)

Align requested chunks with the store’s native chunking to avoid read amplification. Use 'auto' to let xarray choose.

Consolidated metadata#

consolidated=True (the default) reads .zmetadata in a single request rather than fetching per-chunk metadata files. Disable only for stores that lack a consolidated metadata file:

ds = loader.load_data('model.zarr', consolidated=False)

ERA5 on cloud (no CDS account)#

The era5gcs:// and era5aws:// URI schemes access public ERA5 cloud archives without a CDS account:

  • GCS: ARCO-ERA5 Zarr at gs://gcp-public-data-arco-era5/ (requires gcsfs)
  • AWS: ERA5-PDS at s3://era5-pds/ (requires s3fs)

Variable names follow CMIP6/CF conventions (tas, pr, rsds, …) and are mapped to ERA5 internal names automatically. Use --era5-varlist gcs or --era5-varlist aws to see the mapping table.

Programmatic use#

Basic import#

from lib.data_loader import DataLoader, load_model_data, load_observation_data

# Convenience functions (create a fresh loader each call)
ds = load_model_data('/data/model/', variable='ssh',
                     time_range=('2020-01-01', '2020-12-31'),
                     domain={'lon_min': -10, 'lon_max': 5,
                             'lat_min': 50, 'lat_max': 60})

# Reusable loader instance (shares CoordinateMapper, honours cache_dir)
loader = DataLoader(cache_dir='/data/cache')
ds = loader.load_data('copernicus://cmems_obs-sst_atl_phy_nrt_l3s_P1D-m',
                      variable='sea_surface_temperature',
                      domain={'lon_min': -5, 'lon_max': 15,
                              'lat_min': 50, 'lat_max': 65},
                      time_range=('2023-01-01', '2023-01-31'))

Load from YAML config block#

DataLoader.load_from_config() reads an observations: block from a parsed YAML config and dispatches to the appropriate loader:

import yaml
from lib.data_loader import DataLoader

with open('my_config.yaml') as fh:
    cfg = yaml.safe_load(fh)

ds = DataLoader.load_from_config(cfg, section='observations')

Supported source: values in the config block: copernicus, cmip6, isimip, ices, era5gcs, era5aws, auto (local/cloud file, the default).

Example config block for Copernicus:

observations:
  source: copernicus
  dataset_id: cmems_obs-sst_atl_phy_nrt_l3s_P1D-m
  variables: [sea_surface_temperature]
  domain:
    lat_min: 50
    lat_max: 65
    lon_min: -15
    lon_max: 15
  time_range: [2020-01-01, 2020-12-31]

File metadata#

from lib.data_loader import get_file_info

info = get_file_info('/data/model.nc')
# Returns dict with: path, variables, dimensions, coordinates,
#                    time_steps, time_start, time_end, time_resolution, domain

Caching#

Pass cache_dir to DataLoader to cache downloaded remote datasets as compressed NetCDF. On subsequent calls with the same parameters the local cache file is used instead of re-downloading:

loader = DataLoader(cache_dir='/data/cache')

Cache files are named by an MD5 hash of the source identifier, variable, domain, and time range, so different parameter combinations never collide.
