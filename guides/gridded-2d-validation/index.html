Gridded 2D Validation Guide#

Overview#

gridded-2d-validation compares any horizontal 2-D model field against a gridded observation product. Three layer modes are supported:

LayerDescriptionTypical use
surface2-D field or top of 3-D outputSST, SSS, SSH
bottomDeepest wet cell of 3-D outputBottom temperature / salinity
<depth_m>Horizontal slice at a fixed depth (m)Depth-layer climatology

Multiple layers can be run in a single invocation.

Prerequisites#

  • Model NetCDF output accessible as a file-glob pattern
  • A gridded observation file (local or Copernicus Marine / WOA)
  • pip install -e . with the ocean-stats package

Recommended observation sources#

SSTOSTIA (CMEMS)METOFFICE-GLO-SST-L4-REP-OBS-SST (analysed_sst)

OSTIA SST is delivered in Kelvin — add offset: -273.15 in the obs config block.

Config file#

# config/gridded_2d_validation.yaml
area: NS
experiment: Baseline
years: [2010, 2011, 2012]

model:
  base_path: /data/model/NS/
  filename_pattern: "getm_{year}*.nc"
  variable_map:
    temp: sst        # rename model variable if needed

variables: [temp]

observations:
  temp:
    - name: OSTIA
      path:
        source: copernicus
        dataset_id: METOFFICE-GLO-SST-L4-REP-OBS-SST
        variable: analysed_sst
      offset: -273.15

processing:
  layers: [surface, bottom]     # or a single string / list including depths
  chunking:
    time_chunk: 30              # days per processing chunk

output:
  base_dir: ./analyses

Config file on GitHub: config/gridded_2d_validation.yaml

Usage#

# Surface only (config default)
gridded-2d-validation --config config/gridded_2d_validation.yaml

# Surface + bottom in one run
gridded-2d-validation --config config/gridded_2d_validation.yaml \
    --layers surface bottom

# Surface + 50 m depth slice + bottom
gridded-2d-validation --config config/gridded_2d_validation.yaml \
    --layers surface 50 bottom

The --layers flag overrides processing.layers in the config.

Output#

Results are written to analyses/areas/<area>/validations/<experiment>/:

plots/<domain>/<model>/<period>/
    surface/        ← time-mean maps, monthly bias maps, scatter plots
    bottom/
    50m/
tables/<domain>/<model>/
    <variable>_validation_statistics.txt

Statistics text files contain overall + monthly RMSE, bias, MAE, and correlation.

Notes#

  • Observations are regridded onto the model grid using bilinear interpolation (xESMF); weight files are cached by MD5 hash under regrid_weights/.
  • For bottom layer: the deepest non-NaN cell is selected per column using sigma-coordinate-aware extraction.
  • For depth slices: the model field is interpolated vertically to the requested depth before regridding.
