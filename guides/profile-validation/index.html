Profile Validation Guide#

Overview#

Five scripts share the same profile validation workflow:

CommandObservation source
argo-profilesArgo floats (Ifremer ERDDAP)
glodap-profilesGLODAP v2.2023 bottle data
wod-profilesWorld Ocean Database
cruise-ctd-profilesICES / EMODnet cruise CTD casts
fixed-platformMoorings and fixed buoy profiles

All five follow the same config structure, output layout, and command-line flags.

Config structure#

Annotated example configs: config/argo_profiles.yaml · config/glodap_profiles.yaml · config/cruise_ctd_profiles.yaml · config/wod_profiles.yaml · config/fixed_platform.yaml

# .stats.yaml (shared identity — set once per project)
metadata:
  author:    "K. Bolding"
  institute: "BB"
  project:   "NorthSea-MFC"
model:
  name:    "pyGETM"
output:
  base_dir: "./analyses"

# config/argo_profiles.yaml (per-run)
area:       NS
experiment: Baseline

domain:
  lon_min: -5.0
  lon_max: 10.0
  lat_min: 50.0
  lat_max: 62.0

years: [2015, 2022]           # or year: 2020 for a single year
variables: [TEMP, PSAL, DOXY] # subset of available variables

model:
  base_path:        /data/model/NS/
  filename_pattern: "ns_{year}.nc"

output:
  prefix: ns_argo             # filename prefix for plots and tables

Required keys: area, experiment, model.base_path, model.filename_pattern.

Common command-line flags#

All five scripts accept:

FlagDescription
--config FILEYAML config file
--area NAMEOverride area
--experiment NAMEOverride experiment
--years START ENDOverride year range
--variables VAR …Override variables
--analyses-dir DIROverride output.base_dir
--no-stagingSkip writing to simulation registry
--save-residualsWrite model-minus-obs residuals parquet (needed for mle-comparison)

Script-specific flags:

  • argo-profiles: no extra flags
  • glodap-profiles: --no-model (skip model comparison, obs diagnostics only)
  • cruise-ctd-profiles: --dataset-name LABEL (tag the statistics section)
  • fixed-platform: --station NAME (select a single named station)

What each run produces#

analyses/<AREA>/validations/<EXPERIMENT>/
├── plots/
│   └── <domain>/              ← physics | bio
│       └── <model>/           ← e.g. pyGETM
│           └── <period>/      ← e.g. 2020 | 2015-2022
│               └── argo/      ← or glodap/, wod/, cruise/, platform/
│                   ├── <prefix>_overview.png
│                   ├── <prefix>_hovmoller_TEMP.png
│                   └── <prefix>_profiles_YYYY.png
└── tables/
    └── <domain>/
        └── <model>/
            └── argo/
                └── <prefix>_profile_validation_statistics.txt

The overview figure typically has 6 panels:

  • float/station/cruise map coloured by platform or time
  • temperature vs depth scatter
  • salinity vs depth scatter
  • T-S diagram
  • mean vertical profiles (normalised)
  • profile or observation count per platform

Running multiple experiments#

Run the script once per experiment, changing experiment: in the config (or passing --experiment). All outputs go into separate subdirectories:

argo-profiles --config config/argo_profiles.yaml --experiment Baseline
argo-profiles --config config/argo_profiles.yaml --experiment ObsKd
argo-profiles --config config/argo_profiles.yaml --experiment HighRes

Generating residuals for MLE comparison#

Pass --save-residuals alongside the normal run. This writes an additional parquet file that mle-comparison consumes:

argo-profiles --config config/argo_profiles.yaml \
    --experiment Baseline --save-residuals
argo-profiles --config config/argo_profiles.yaml \
    --experiment ObsKd    --save-residuals

Residuals are written to:

analyses/areas/<AREA>/validations/<EXPERIMENT>/argo_residuals.parquet

The parquet has columns: time, lat, lon, depth, variable, value (obs), model_value, and optionally profile_id.

See guides/mle-comparison.md for the ranking workflow.

GLODAP-specific notes#

The GLODAP v2.2023 merged master file (~300 MB) is downloaded automatically on first run and cached locally. Set glodap.cache_dir in the config to control where it lands:

glodap:
  cache_dir: /data/obs/glodap/
  qc_flags: [2]   # 2 = good data; 0 includes unqualified

Pass --no-model to produce observation-only diagnostics (station map, T-S diagram, profile climatology) without loading any model output.

Fixed-platform notes#

The fixed-platform script reads from the station catalogue (config/station_catalogue.yaml). Named stations include L4, E1, Stonehaven, Helgoland, and the MARNET and CEFAS SmartBuoy networks. Use download-station to fetch and cache station data before running fixed-platform.

download-station download --station L4 --output-dir /data/stations
fixed-platform --config config/fixed_platform.yaml --station L4
