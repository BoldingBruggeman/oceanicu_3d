Tidal Analysis Guide#

Overview#

tidal-analysis extracts tidal harmonic constituents from model output and validates them against satellite tidal atlases (FES2014, TPXO9) or tide gauge records from the GESLA database.

The harmonic analysis uses utide to fit sinusoidal components at each grid point or station location. Model amplitudes and phases are then compared to the reference atlas or gauge data.

Analysis modes#

ModeFlagDescription
spatial--mode spatialHarmonic analysis on the full model grid, compared to a tidal atlas
gesla--mode geslaHarmonic analysis at GESLA tide gauge locations
both--mode bothRun spatial and GESLA together (default)
stations--mode stationsNamed stations defined in config

Config structure#

Annotated example: config/tidal_analysis.yaml

# config/tidal_analysis.yaml
area:       NS
experiment: Baseline

model:
  base_path:        /data/model/NS/
  filename_pattern: "ns_{year}.nc"

years: [2015, 2022]

# Constituents to analyse (subset of M2, S2, K1, O1, N2, M4, …)
constituents: [M2, S2, K1, O1]

analysis:
  mode: both          # spatial | gesla | both | stations

# Tidal atlas for spatial comparison
atlas:
  type:  fes2014      # fes2014 | tpxo9
  path:  /data/atlas/fes2014/

# GESLA tide gauge database for station comparison
gesla:
  path: /data/gesla/GESLA3.0_ALL.nc
  min_years: 10        # minimum record length to include a gauge

output:
  prefix: ns_tidal

Required keys: area, experiment, model.base_path, model.filename_pattern.

Usage#

# Full run with default mode (both)
tidal-analysis --config config/tidal_analysis.yaml

# Spatial only
tidal-analysis --config config/tidal_analysis.yaml --mode spatial

# GESLA stations only
tidal-analysis --config config/tidal_analysis.yaml --mode gesla

# Single constituent
tidal-analysis --config config/tidal_analysis.yaml --constituents M2

# Parallel processing (uses all CPUs by default)
tidal-analysis --config config/tidal_analysis.yaml --workers 8

# Append to an existing statistics file (useful when running multiple datasets)
tidal-analysis --config config/tidal_analysis.yaml --dataset-name "FES2014" \
    --append-stats

Statistics file format#

The statistics file contains one section per constituent with area-averaged metrics:

M2 Amplitude
  RMSE:        0.032 m
  Bias:       -0.008 m
  Correlation: 0.987

M2 Phase
  RMSE:        4.2 deg
  Bias:        1.1 deg

When --append-stats is used, each call adds a new section labelled with --dataset-name, making it easy to compare multiple atlases or gauge datasets in one file.

Performance#

The spatial mode runs utide.solve() at every model grid point. For large domains this is the bottleneck. Use --workers N to parallelise across grid rows (default: all available CPUs). A typical North Sea grid (200×300 points) with 8 years of hourly output takes 5–15 minutes depending on the number of constituents.

The GESLA mode only analyses at gauge locations (~a few hundred) and is much faster.
