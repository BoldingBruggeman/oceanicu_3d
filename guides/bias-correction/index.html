Bias Correction Guide#

Overview#

The bias correction workflow adjusts CMIP6 (or other) atmospheric forcing fields so their statistical properties match an observational reference (typically ERA5) over a calibration period. The corrected output is used as forcing for future ocean model projections.

Two commands share the same YAML config:

CommandRole
bc correctApply bias correction → write NetCDF output
bc diagnosticsInspect the correction → write maps, seasonal cycles, trend plots, statistics table

Both can also be called via the aliases bc-correct and bc-diagnostics.

Correction methods#

MethodFlagDescription
Delta (additive)deltaShift future by the calibration-period mean bias
Quantile mappingqmMap quantiles of historical onto reference quantiles
Quantile delta mappingqdmPreserve relative future trends while correcting distribution

qdm is the default and is recommended for most variables. delta is fast but only corrects the mean; qm corrects the full distribution but does not preserve climate change trends.

Config structure#

Fully annotated example: config/bias_correction_example.yaml

# config/bc.yaml

metadata:
  author:     "K. Bolding"
  institute:  "BB"
  project:    "NorthSea-MFC"
  area:       NS
  experiment: ssp245
  scenario:   ssp245      # tag used in output filenames

domain:
  lon_min: -5.0
  lon_max: 10.0
  lat_min: 50.0
  lat_max: 62.0

calibration:
  start: "1990-01-01"
  end:   "2014-12-31"

future:
  start: "2015-01-01"
  end:   "2100-12-31"

correction:
  method:     qdm
  n_quantiles: 100

variables:
  - name: tas
    kind: additive          # additive or multiplicative
    regridding:
      method: bilinear
  - name: pr
    kind: multiplicative
    regridding:
      method: bilinear

# Observational reference (ERA5 or equivalent)
reference:
  tas:
    source: local
    path: /data/era5/tas_{year}.nc
    variable: t2m
  pr:
    source: local
    path: /data/era5/pr_{year}.nc
    variable: tp

# CMIP6 historical
historical:
  tas:
    uri: "cmip6://MPI-ESM1-2-HR/historical/tas/day/r1i1p1f1"
  pr:
    uri: "cmip6://MPI-ESM1-2-HR/historical/pr/day/r1i1p1f1"

# CMIP6 future scenario
future_model:
  tas:
    uri: "cmip6://MPI-ESM1-2-HR/ssp245/tas/day/r1i1p1f1"
  pr:
    uri: "cmip6://MPI-ESM1-2-HR/ssp245/pr/day/r1i1p1f1"

output:
  dir: ./bc_output/{scenario}
  filename_pattern: "{variable}_bc_{year}.nc"
  compress: true
  clim_period: 1            # 1 = one file per year; >1 = multi-year means

cache:
  enabled: true
  dir: ./regrid_weights/

Running the correction#

# Full run (correction + disaggregation if configured)
bc correct --config config/bc.yaml

# Single variable
bc correct --config config/bc.yaml --variable tas

# Override method
bc correct --config config/bc.yaml --method qdm

# Override scenario (useful for running ssp245/ssp585 from the same base config)
bc correct --config config/bc.yaml --scenario ssp585

# Dry run — shows which files would be read and written, no data loaded
bc correct --config config/bc.yaml --dry-run

Stages#

bc correct runs two stages by default. Use --steps to control which run:

# Correction only (skip temporal disaggregation)
bc correct --config config/bc.yaml --steps correct

# Disaggregation only (post-process previously corrected files)
bc correct --config config/bc.yaml --steps disaggregate

Disaggregation (optional) distributes daily corrected output to sub-daily resolution using a diurnal climatology derived from high-frequency ERA5. It requires a disaggregation: block in the config with per-variable ERA5 paths and smooth_window / clim_years settings.

Output files#

One NetCDF per variable per year (or per multi-year window if clim_period > 1):

bc_output/ssp245/
    tas_bc_2015.nc
    tas_bc_2016.nc
    …
    tas_bc_2100.nc
    pr_bc_2015.nc
    …

File naming is controlled by output.filename_pattern. The {scenario} and {method} placeholders are substituted automatically.

Running diagnostics#

Diagnostics read the same config and the corrected output files produced above:

bc diagnostics --config config/bc.yaml

# Single variable
bc diagnostics --config config/bc.yaml --variable tas

# Write to a specific directory
bc diagnostics --config config/bc.yaml --output-dir ./diag/ssp245/

# Dry run — shows input and output paths only
bc diagnostics --config config/bc.yaml --dry-run

Diagnostic output#

Results land under analyses/areas/<area>/scenarios/<scenario>/:

analyses/areas/NS/scenarios/ssp245/
└── tas/
    ├── plots/
    │   ├── calibration_maps_ssp245_bilinear.png   — ref | model raw | bias
    │   ├── seasonal_cycle_ssp245_bilinear.png     — monthly climatology comparison
    │   ├── future_trend_ssp245_bilinear.png       — annual means + linear trend
    │   └── monthly_breakdown_ssp245_bilinear.png  — decadal heatmap
    └── tables/
        └── statistics_ssp245_bilinear.txt         — per-month RMSE / bias / corr

Calibration maps#

Three-panel map: observational reference mean | CMIP6 historical raw mean | bias (raw − reference). Both field panels share one colour scale; the bias panel uses a diverging scale.

Seasonal cycle#

Monthly area-mean climatology: reference (black), model historical raw (blue dashed), and the corrected future split by decade (colour ramp). If the future scenario has no inter-annual variability a single line is shown with an explanatory note.

Future trend#

Annual area-mean of the corrected output over the full future period with a linear trend line and trend rate per decade.

Statistics table#

Per-month and overall RMSE, mean bias, MAE, and Pearson correlation between the raw CMIP6 historical and the reference over the calibration period, plus a full error distribution summary.

Multiple scenarios#

Run bc correct and bc diagnostics once per scenario, overriding --scenario:

for scenario in ssp245 ssp370 ssp585; do
    bc correct     --config config/bc.yaml --scenario $scenario
    bc diagnostics --config config/bc.yaml --scenario $scenario
done

The scenario tag appears in all output paths and filenames so runs do not overwrite each other.

Performance notes#

  • Correction uses Dask for lazy loading. Memory stays bounded because output is written year by year.
  • Diagnostics use a fast ERA5 loading path (time-only chunks, no spatial tiling) — typically ~2 s/year instead of ~60 s/year with the correction path.
  • Regridding weight files are cached under cache.dir; the cache is reused across all variables and scenarios with the same grids.
